# Design and replace character(s) for a mini-game

### Why this task matters
These tips build on what you already know from working with characters in intro and outro scenes. In mini-games, the characters often do more. They may be moving, attacking, or interacting with puzzles and NPCs so careful setup is essential. Here’s what to consider: Revisit the [Design and replace character(s) for intro & outro](https://github.com/endlessm/threadbare/discussions/602)
### Steps
### Git Setup
1. **Update your local repository.** Open Git Bash and navigate to your project folder. Before creating a new branch, switch back to the main branch and pull the latest changes from your team's GitHub repository. This makes sure that the new branch you are about to create will have the latest updates from your team's StoryQuest.
     ```bash
     git switch main
     git pull
     ```
2. **Create a new Git branch**
     Now, create a new branch for your work:
     ```bash
     git switch -c new-branch-name
     ```
     In general, choose a branch name that reflects what you're doing, like `intro-outro-tilemap` or `add-custom-path`. This keeps your work organized.  [Click here for a more detailed guide and more information on updating your local repository and creating a branch.](https://github.com/endlessm/threadbare/discussions/738)
---

**2. Copy & Load New SpriteFrames**
You don’t need to make a brand new player or enemy from scratch. Start by copying the template SpriteFrames .tres file from any character you want to modify.

Method 1
- Rename it using your StoryQuest name (e.g., stella_spriteframes.tres).
- Load your copy into the character’s Sprite Frames field in the Inspector.
- You’ll now see a list of animations (like idle, walk, etc.). You can either:
Drag image frames from your spritesheet .png in the FileSystem into the correct animation row in the Inspector

Method 2
- Open your new .tres and use the Sprite Frames panel (bottom center of the Godot editor) to add frames using the “Add from Sprite Sheet” (waffle or grid) button.


**3. Know Which Animations You Need**
Each mini-game only uses certain animations. Focus on making these match your story’s look: 
- idle and walk: Needed in all mini-games.
- Attack: Only needed in combat
- Defeated: Only needed in combat and stealth

You can leave unused animations empty just don’t delete or rename the animation track itself.
Keeping the animation track names (like walk, idle, attack) the same is essential. These names are referenced in scripts and won’t work if renamed.

**4. Adjust the Properties of the Character in the Inspector.**
Click on the character node in the Scene Tree. The Inspector now shows settings specific to this copy of the character. Adjustments here only affect this version.
A few example properties:
- Scale – resize the character.
- Position – fine-tune where they appear.
- Rotation – flip or angle if needed.
This is also where you’ll change out things like:
- The NPC name (for talkers).
- The dialogue file they use.
- The sprite frames resource.
This method allows you to fully customize the look, feel, and function of a character in your mini-game without affecting other scenes.

**5. (Advanced) Customize Internal Parts With “Make Unique”**
If you need to adjust something inside the character, like the collision shape or interact area, follow these steps so your change applies only to your scene:
- Right-click the character node and choose “Editable Children.”
- Find and select the child node you want to change (e.g., CollisionShape2D).
- In the Inspector, click the arrow next to the property you want to change.
- Choose “Make Unique.”

Now, edits will apply only to this copy. This avoids accidentally breaking other characters that use the same base setup.

6. Save your changes. Use Scene → Save All Scenes.
---
### Save To Git and GitHub
7. **Stage and commit your changes.** In Git Bash, here is an example of how this flow will generally look:
     ```bash
     git status
     git add scenes/quests/story_quests/yourStoryQuest/
     git commit -m "Add your message here - explain to your team what you changed"
     git push --set-upstream origin your-branch-name
     ```
- Make sure the file path shows your actual story quest name, write your own descriptive commit message, and use your actual branch name.
- You can use `git add` to add more than one file simply by putting a space between file paths you paste in.
- You only need to use `--set-upstream` the first time you push a new branch to GitHub.
 [Click here for a more detailed guide and more information on saving and pushing your changes to GitHub.](https://github.com/endlessm/threadbare/discussions/816)
8. **Submit a pull request.** Go to GitHub, navigate to your fork, and click **Compare & pull request**. Write a clear title and description indicating what you changed and why, then click **Create pull request**.
[Click here for a more detailed guide and more information on creating a pull request to your team's forked repo.](https://github.com/endlessm/threadbare/discussions/826)

**Note:** Pull requests are better with a peer review! Click [here](https://github.com/endlessm/threadbare/discussions/859) to learn more. If you are working alone, you can merge your own pull request after submitting it.
