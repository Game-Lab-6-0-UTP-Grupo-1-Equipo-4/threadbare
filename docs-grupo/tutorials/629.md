# Add narrative text for a mini-game

### Why this task matters
This is where your story comes alive. The narrative helps your player understand the journey and its emotional stakes. Revisit the [Add narrative text for intro & outro](https://github.com/endlessm/threadbare/discussions/604) tutorial if needed. 
### Git Setup
1. **Update your local repository.** Open Git Bash and navigate to your project folder. Before creating a new branch, switch back to the main branch and pull the latest changes from your team's GitHub repository. This makes sure that the new branch you are about to create will have the latest updates from your team's StoryQuest.
     ```bash
     git switch main
     git pull
     ```
2. **Create a new Git branch**
     Now, create a new branch for your work:
     ```bash
     git switch -c new-branch-name
     ```
     In general, choose a branch name that reflects what you're doing, like `intro-outro-tilemap` or `add-custom-path`. This keeps your work organized.  [Click here for a more detailed guide and more information on updating your local repository and creating a branch.](https://github.com/endlessm/threadbare/discussions/738)

### Tips
When adding story and dialogue to a mini-game scene, there are a few important differences from how it worked in the intro and outro scenes:

**3. Starting Dialogue Is Handled Differently Per Mini-Game**
- Stealth & Sequence Mini-Games: These usually use a Cinematic node at the beginning of the scene, just like intro scenes. You can add your starting dialogue here.
- Combat Mini-Game: There is no Cinematic node. Instead, the starting dialogue is placed inside the FillGameLogic node. This is done intentionally to make sure the dialogue appears before the action starts. So if you're designing your own version of this game, always add your opening narrative inside FillGameLogic.

**4. Dialogue After Completing the Game**
All three mini-game templates use a CollectibleItem node to signal the end of the mini-game (your “completion” condition). This node also includes a dialogue field that is shown after the player completes the mini-game. Be sure to check and customize this dialogue to match your story.

**5. Optional Checkpoint Dialogue (Stealth Only)**
In the stealth mini-game, when the player reaches a checkpoint, a bit of extra dialogue can appear. You can find this by clicking on the Checkpoint node in the Scene Tree and adjusting the dialogue in its Inspector. This is a great place to give encouragement, add a hint, or continue the story as the player gets closer to the goal.

### Save To Git and GitHub
6. **Stage and commit your changes.** In Git Bash, here is an example of how this flow will generally look:
     ```bash
     git status
     git add scenes/quests/story_quests/yourStoryQuest/
     git commit -m "Add your message here - explain to your team what you changed"
     git push --set-upstream origin your-branch-name
     ```
- Make sure the file path shows your actual story quest name, write your own descriptive commit message, and use your actual branch name.
- You can use `git add` to add more than one file simply by putting a space between file paths you paste in.
- You only need to use `--set-upstream` the first time you push a new branch to GitHub.
 [Click here for a more detailed guide and more information on saving and pushing your changes to GitHub.](https://github.com/endlessm/threadbare/discussions/816)

7. **Submit a pull request.** Go to GitHub, navigate to your fork, and click **Compare & pull request**. Write a clear title and description indicating what you changed and why, then click **Create pull request**.
[Click here for a more detailed guide and more information on creating a pull request to your team's forked repo.](https://github.com/endlessm/threadbare/discussions/826)

**Note:** Pull requests are better with a peer review! Click [here](https://github.com/endlessm/threadbare/discussions/859) to learn more. If you are working alone, you can merge your own pull request after submitting it.
